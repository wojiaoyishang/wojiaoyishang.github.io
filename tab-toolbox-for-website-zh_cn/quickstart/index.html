Tab Toolbox for Website 快速入门

安装

克隆仓库

克隆仓库，方法不唯一，例如使用 Git 命令：

$ git clone git@github.com:wojiaoyishang/Tab-Toolbox-for-Website.git

安装 Python 运行环境

克隆仓库完成之后，您需要检查您的电脑中是否存在 Python 环境，因为程序基于 Python 运行。请注意您的 Python 版本必须在 3.8.0 及以上，3.7.X 的 Python 版本并未进行测试。

Python 的安装地址可以前往 Python 官网下载，链接在这里：https://www.python.org/downloads/release/python-380/

安装 Python 的模块

在确认 Python 的环境准确无误后，请确认 Python 中所支持的模块是否安装。对于 Tab Toolbox for Website 运行必要的模块在 仓库根目录下的 requirements.txt 中。内容如下：

bs4==0.0.1
Flask==2.1.3
gevent==20.6.2
MarkupSafe==2.1.1
pandas==1.4.3
psutil==5.7.2
requests==2.24.0
lxml==4.5.2

可以使用 pip 进行安装，如下：

$ pip install -r requirements.txt

或者您可以试试，

$ python -m pip install -r requirements.txt

启动

直接启动

在确保准确无误安装好 Python 以及其 模块 后，可以进行运行测试。使用 Python 启动 main.py 。

$ python main.py

您可以根据程序输出的提示简单配置 Tab Toolbox for Website 。随后配置将会写入 Tab Toolbox for Website 根目录下的 config.py 中。并在下次直接启动时不会再次要求配置。配置引导仅有中文，下面是对应的翻译......

命令行启动

我们除此之外还提供了命令行启动的方式，传入 -h 可查看帮助。

python main.py -h
Usage: main.py [options]

Options:
  -h, --help            show this help message and exit
  -H host, --host=host  Host for Running
  -p port, --port=port  Port for Running
  -D, --development     Development Mode
  -d, --debug           Debug Mode

下面是启动的示例：

(1) 绑定 0.0.0.0:8080 ，您可以使用

注意：请不要随意将 IP 绑定在 0.0.0.0 上，这可能会导致公网访问的风险。

$ python main.py -H 0.0.0.0 -p 8080

或者使用

$ python main.py --host=0.0.0.0 --port=8080

(2) 打开开发模式与 Debug 模式

注意：开发者模式与Debug模式，必须存在IP地址与端口参数。

$ python main.py --host=0.0.0.0 --port=8080 -D -d

或者使用

$ python main.py --host=0.0.0.0 --port=8080 --development --debug

需要注意的是，使用命令行启动时会自动改写配置文件，即使您下次在使用直接启动的方式，也会直接使用上一次命令行启动时的配置。关于配置文件，下面会加以说明。

关于配置文件

配置文件 config.py 的内容如下：

# 注意此文件仅用于配置程序的一些内置内容，比如运行模式、版本号等。按理来讲在发布时此文件已经设定好，不需要更改。

DEVELOPMENT = False  # 如果更改这个变量那么就会打开开发模式，而不是发布模式
DEBUG = False   # 这个是否为 Debug 模式，主要控制的是 app.run() 中的设置，若是命令行启动则可以忽略
HOST = "127.0.0.1"  # 设置运行 IP
PORT = 99  # 运行端口
FIRST_RUNNING = False  # 是否第一次运行

TAB_TOOLBOX_VERSION = "1.0.0.1"

开发模式打开时，会使用 flask 中的 app.run() 函数运行程序，而不是 WSGIServer 。

Debug 模式必须在 开发模式 打开的前提下才能使用，这会让 app.run() 的 Debug 参数为 True 。
